Welcome to a new series about MCP Apps — a brand new extension to the Model Context Protocol that lets tools return interactive UI instead of just text, and how Claude Desktop specifically supports it. What this series is about

MCP has always been about connecting a model to tools and data through text. MCP Apps is new: it lets an MCP server hand back a whole interactive UI that renders live inside the chat window.

This series is about understanding that extension, and specifically how Claude Desktop implements it — with a real, working example built from scratch along the way.

Up to this point, MCP has mostly meant: the model calls a tool, the tool returns text or structured data, and that's what shows up in the conversation. MCP Apps changes that by letting a tool point at an actual interactive HTML interface — dashboards, forms, 3D viewers — that the host renders live. This series exists to understand that shift, and in particular how Claude Desktop, specifically, supports it today.

Two things that sound alike but aren't

MCPB
How you install an MCP server into Claude Desktop. A zip file, one-click setup, runs locally.
MCP Apps
What happens after a tool call — a server returns UI, the host renders it inline.
Here's the distinction that's easiest to conflate, and worth locking in early. MCPB is how you install an MCP server into Claude Desktop — settings, extensions, a dot m c p b file. MCP Apps is a completely different layer: what happens after a tool call, when the server hands back an interactive UI instead of plain text. A single server can use both — ship as an MCPB for easy install, and also expose tools that render MCP Apps UI. They're orthogonal, not the same thing.

Glossary — the terms we'll keep using

MCP
Model Context Protocol — connects an LLM host to external tools, data, and prompts.
MCP Apps
SEP-1865, shipped Jan 2026 — lets a tool point at a ui:// resource the host renders.
Host
The client app embedding the UI — Claude Desktop, claude.ai, VS Code, etc.
App Bridge
SDK module hosts use to render apps in sandboxed iframes and manage messaging.
MCPB
Claude Desktop's packaging format for installable local MCP servers (formerly "DXT").
A handful of terms will come up again and again, so let's define them once. M C P is the base protocol. MCP Apps is the specific extension this series is about. Host just means whatever app is embedding the UI — Claude Desktop, claude dot A I, VS Code. App Bridge is the SDK piece hosts use to actually render an app safely. And MCPB is Claude Desktop's install format, previously called D X T.
